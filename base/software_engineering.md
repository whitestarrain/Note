# 概述

软件工程是应用系统化、规范化的方法来开发、运行和维护软件的学科。

# 可行性研究

从技术可行性、经济可行性和操作可行性三方面评估项目是否值得开发。

# 需求分析

通过与用户沟通，明确系统必须做什么（功能需求）和做到什么程度（非功能需求）。

# 形式化说明技术

使用数学化的符号和方法精确描述软件系统的功能和行为规约。

# 总体设计

将需求分析的结果转化为软件的体系结构，划分模块并定义模块间接口。

# 详细设计

为每个模块确定具体的算法和数据结构，产出可直接编码的设计方案。

# 实现与测试

编码实现设计方案，并通过单元测试、集成测试、系统测试等手段验证正确性。

# 维护

软件交付后的修改活动，包括纠错性维护、适应性维护、完善性维护和预防性维护。

# 面向对象

运用面向对象分析（OOA）、面向对象设计（OOD）方法进行软件开发。

# 软件项目管理

涉及项目计划、进度管理、风险管理、质量保证和配置管理等活动。
